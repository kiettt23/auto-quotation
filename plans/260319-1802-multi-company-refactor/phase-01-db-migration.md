# Phase 1: DB Migration

## Priority: P1 | Status: completed | Effort: 2h

## Overview

Add `userId` column to tables currently isolated by `companyId`. Add new fields to company and customer. Migrate existing data.

## Key Insights

- Currently 6 tables reference `companyId`: customer, product, category, unit, document, document-type
- After refactor: customer, product, category, unit, document-type switch to `userId`. Document keeps `companyId` (FK to know which company issued it)
- Company table: add `userId` (rename from `ownerId`), add `driverName`, `vehicleId`
- Customer table: add `deliveryName`, add `userId`, drop `companyId` FK

## Schema Changes

### company table
```
KEEP: id, name, address, phone, taxCode, email, bankName, bankAccount, logoUrl, headerLayout, createdAt, updatedAt
RENAME: ownerId → userId (or add userId, drop ownerId)
ADD: driverName text, vehicleId text
REMOVE: nothing (keep all existing fields)
```

### customer table
```
ADD: userId text NOT NULL (populate from company.ownerId via companyId join)
ADD: deliveryName text
DROP: companyId FK (after data migration)
```

### product table
```
ADD: userId text NOT NULL
DROP: companyId FK
```

### category table
```
ADD: userId text NOT NULL
DROP: companyId FK
```

### unit table
```
ADD: userId text NOT NULL
DROP: companyId FK
```

### document_type table
```
ADD: userId text NOT NULL
DROP: companyId FK
UPDATE: unique constraint uq_document_type_company_key → uq_document_type_user_key (userId, key)
```

### document table
```
KEEP: companyId FK (documents belong to a company)
ADD: userId text NOT NULL (for filtering "my documents")
```

## Implementation Steps

1. Update all schema files with new columns (add `userId`, keep `companyId` temporarily nullable where dropping)
2. Run `npx drizzle-kit generate` to create migration SQL
3. Review generated SQL, add data migration statements:
   ```sql
   -- Populate userId from company.ownerId for all child tables
   UPDATE customer SET user_id = (SELECT owner_id FROM company WHERE company.id = customer.company_id);
   UPDATE product SET user_id = (SELECT owner_id FROM company WHERE company.id = product.company_id);
   -- ... same for category, unit, document_type, document
   ```
4. After userId populated, drop companyId columns from customer/product/category/unit/document_type
5. Rename company.ownerId → company.userId
6. Run `npx drizzle-kit push` against staging DB
7. Verify data integrity with SQL queries

## Risk

- **Migration order matters**: must add userId + populate BEFORE dropping companyId
- **Staging DB has existing data**: need careful data migration, not just schema change
- **FK constraints**: drop FKs before dropping columns
